Password refused

If Maintenance refuses your password, verify or try the following operations.

Maintenance runs only on an Administrator accounts. Verify that you can really administer the computer in System Preferences > Accounts > Password tab.

Verify that the Shift key is not pressed.

Verify the configuration of your keyboard (QWERTY, QWERTZ, AZERTY, and so on).

Insert the Install Disc 1 and restart while pressing the C key. Select your language then click the Disk Utilities in the Utilities menu. Select your startup disk, and click the Repair Disk button. When done, click the Repair Disk Permissions button. Then restart normally, open Maintenance, and try again...

If Maintenance again refuses your password...

Verify the presence of the "StandardAdditions.osax" file in the /System/Library/ScriptingAdditions/ folder.

If there is another "StandardAdditions.osax" file in the /Library/ScriptingAdditions/ folder, remove it.

Restart the computer, then open Maintenance.

If Maintenance always refuses your password, starting in Safe Mode (Safe Boot) can sometimes correct the problem.
Warning: Some features don't work in Safe Mode. For example, you can't use DVD Player, capture video in iMovie, use an AirPort card, use audio input or output devices, or use an internal or external USB modem.

Be sure the computer is turned off.

Press the power button.

Immediately after you hear the startup tone, press and hold the Shift key. During the startup, you will see "Safe Boot" on the login window, which appears even if you normally log in automatically.

To leave Safe Mode in any version of Mac OS X, restart the computer normally, without holding any keys during startup.

Open Maintenance.
