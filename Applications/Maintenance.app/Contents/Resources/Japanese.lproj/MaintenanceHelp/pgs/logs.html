Logs and CrashReporter

When an application is open, it sends messages between the application, system, and its components. This information is stored in log files. The system also creates logs and archives them during System maintenance operations. When an application does not respond, the details are saved in a Log file that you can use to diagnose the problems, or to send it (if necessary) to the developer. This option deletes any Diagnostic Reports created by the system and relaunches the syslogd daemon (process that records the activities of the system).

Maintenance allows you to remove these logs, the compressed archives, the CrashReporter, and the following web browsers' History: Amaya, BumperCar, Camino, Chromium, Cruz, Demeter, DeskBrowse, Desktop Web Browser, Dreams, Fake, Firefox, Flock, Gmail Browser, Google Chrome, Hana, iBrowser, iCab, Internet Explorer, iTubeX, KidsBrowser, KidZui, Lightbrowser, Madfox, MiaBrowser, MiniBrowser, NagaraBrowser, Navigateur Web, Navigator (Netscape), Nova X, OmniWeb, Opera, Radon, Roccat, Safari, Scavenger, Scourge, SeaMonkey, Shiira, SRWare Iron, Stainless, Sunrise, surfDude, TenFourFox, TrailBlazer, WamCom, WebBackdrop, WebDesktop, WebKit, Webstractor, wKiosk, Wyzo, and Xyle scope.

Note: The Software Update Log is not deleted.
