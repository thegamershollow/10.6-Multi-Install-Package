验证并修理权限

Mac OS X 包括了一个基于类 UNIX 的子操作系统并在文件系统使用权限功能。磁盘上的每个文件和文件夹都设置了权限如可读，可写，或可执行。

很多装进 Mac OS X 的程序都使用了一种安装包文件 (.pkg 文件)。每次从安装包进行安装时，一个 "原料清单" 文件 (.bom 文件) 会储存在安装包的 receipt 文件中 (位于 /Library/Receipts/)。如果您查看 Receipts 文件夹，您可以看到所有的以 .pkg 为扩展名的文件，包括 Mac OS X 安装时的文件 (比如，BaseSystem.pkg)。不用担心，这些文件占不了多大空间您也不必把它们扔进废纸篓。每一个 .bom 文件都包含了安装包安装的文件列表和每个文件应有的权限。

当您使用 OnyX 验证并修理磁盘权限，就会开始复查 /Library/Receipts/ 下的 .bom 文件并比较每个文件的权限。如果权限不一致，OnyX 就会修正差异。如果您从 /Library/Receipts/ 移去了这些安装清单文件，被这些安装包安装的文件就无法确定其权限，这就是为什么不能从 /Library/Receipts/ 删除文件的原因。

OnyX 会修理这些文件的权限但不会修理您个人文件夹下文稿的权限...

要修理权限：

运行 OnyX 并输入管理员密码。

打开 "维护" 面板并点按 "权限" 标签页。

点按 "执行" 按钮.

一旦修理结束，结果就会显示在文本区。一般来说，显示的只有信息通知。未修复的不正确权限将不会对您的电脑功能造成影响。
