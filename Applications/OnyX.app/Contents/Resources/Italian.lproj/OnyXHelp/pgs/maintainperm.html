Verifica e ripara i Permessi

Mac OS X incorpora un sottosistema basato su un sistema operativo in stile UNIX che utilizza i privilegi nel file system. A ogni documento e cartella sul disco rigido è associato un insieme di privilegi che definisce gli utenti autorizzati a leggere, sovrascrivere o eseguire il documento o la cartella.

Molte delle applicazioni che installate in Mac OS X provengono da files package (.pkg files). Ogni volta che si installa qualcosa da un package, un file "Elenco del Contenuto" (file .bom) è immagazzinato in una cartella del Sistema o in un file di ricevuta (.pkg). Ognuno di quei file .bom contiene una lista dei files installati da quel package, e gli specifici permessi per ogni file.

Quando si usa OnyX per verificare e riparare i permessi disco, egli compara ogni file .bom e compara la lista in esso contenuta con gli attuali permessi di ogni file elencato. Se i permessi sono diversi, OnyX corregge le differenze. Se rimuovete le ricevute o i files .bom dalla cartella /Libreria/Receipts/, i files installati dai packages non potranno essere né verificati né riparati. Ecco perché non dovreste mai cancellare niente dalla cartella dalla cartella /Libreria/Receipts/.

OnyX ripara i permessi di questi files ma non riparerà i permessi dei documenti nella vostra cartella Inizio.

Per riparare i permessi:

Lanciate OnyX. Vi chiederà la password amministratore.

Aprite la finestra "Manutenzione" e cliccate sul pannello "Permessi".

Cliccate sul pulsate "Esegui".

Una volta terminata la riparazione, il risultato sarà visualizzato nel campo di testo presente per questo scopo. Di solito, i messaggi di avviso hanno solo scopo informativo. I permessi errati che non sono stati riparati non avranno conseguenze negative sul funzionamento del vostro computer.

• Messaggi che è possibile ignorare

Nota: Anche la finestra Automatizza permette di riparare i permessi, ma non visualizzerà nessun risultato.
