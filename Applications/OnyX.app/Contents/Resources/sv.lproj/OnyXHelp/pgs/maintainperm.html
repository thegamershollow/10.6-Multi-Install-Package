Reparera skivbehörigheter

Mac OS X innehåller ett undersystem baserat på UNIX. I detta system så har varje fil och mapp på din hårddisk behörigheter tilldelade till sig som bestämmer vem som kan läsa, skriva eller exekvera.

Många objekt du installerar i Mac OS X installeras från paketfiler (.pkg). Varje gång något installeras från ett paket, skapas och lagras en fil (.bom) i systemet eller i en receptfil (.pkg). Varje .bom fil innehåller en lista av filer som installerades av det paketet och de korrekta behörigheterna för varje fil.

När du använder OnyX för att kontrollera och reparera skivbehörigheter, så kollas varje .bom fil och jämför dess lista med de aktuella behörigheterna för varje listad fil. Om behörigheterna inte matchar, så korrigerar OnyX detta. Om du tar bort recept eller .bom filer från /Library/Reciepts/, så kan inte filerna som paketet installerat verifieras eller repareras. Därför bör du inte radera filer från /Library/Reciepts/.

OnyX reparerar behörigheterna för dessa filer men reparerar inte behörigheter för dokument i din hemmapp.

För att reparera behörigheter:

Kör OnyX, mata in ditt administratörslösenord.

Öppna panelen "Underhåll" och klicka på fliken "Skivbehörigheter".

Klicka på knappen Genomför.

När reparationen är genomförd, så visas resultatet i textfältet för detta ändamål. Generellt sett, så är varningsmeddelandena bara informationsmeddelanden. Felaktiga behörigheter som inte reparerats har ingen inverkan på din dators funktion.

• Meddelanden som du kan ignorera

Notera: Panelen Automatisera tillåter dig också att reparera skivbehörigheter men visar inte resultatet.
