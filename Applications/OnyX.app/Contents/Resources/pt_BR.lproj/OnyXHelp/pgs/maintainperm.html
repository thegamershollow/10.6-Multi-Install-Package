Verificar e Reparar Permissões

O Mac OS X agrega um subsistema baseado em um sistema operacional Unix. Nesse tipo de sistema, todo arquivo e pasta no disco tem a si associado um conjunto de permissões, determinando quem pode ler, gravar ou executar tal item.

Muitas das coisas que você instala no Mac OS X vêm de arquivos de pacotes (.pkg ), que geram um arquivo (.bom) guardado em uma pasta do sistema ou em um arquivo de recibo (.pkg). Cada um desses arquivos .bom contêm uma lista de arquivos instalados pelo pacote, com as permissões corretas de cada item instalado.

Ao usar o OnyX para verificar e reparar as permissões do disco, cada arquivo .bom é analisado e comparado à lista com as devidas permissões de cada arquivo. Se as permissões forem divergentes, OnyX as corrigirá. Ele reparará as permissões desses arquivos mas não dos documentos em sua pasta pessoal.

Caso você remova os recibos ou os arquivos .bom de /Biblioteca/Recibos/, os arquivos instalados pelos pacotes não poderão ser verificados nem reparados. Este é o motivo pelo qual você não deve apagar nada da pasta /Biblioteca/Receipts/.

Para reparar as permissões:

Abra o OnyX. Ele pedirá sua senha de administrador.

Abra o painel Manutenção, e selecione a aba Permissões.

Clique o botão Executar.

Após o término do reparo, o resultado será mostrado no campo providenciado para este propósito. Normalmente, os alertas mostrados são apenas informativos. Permissões incorretas que não sejam reparadas não terão qualquer consequência no funcionamento do seu computador.

• Menssagens que podem ser ignoradas com segurança

Obs: Você pode usar o painel Automação para reparar as permissões, mas ele não lhe mostrará os resultados.
